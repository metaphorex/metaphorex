## What It Brings

A spider's web: interconnected strands radiating from and between anchor
points, any point reachable from any other by traversing the structure.
Tim Berners-Lee mapped this topology onto a system of hyperlinked
documents, and the name stuck so completely that "the web" now means
the internet to most people (even though it is only one layer of it).

- **Topology as structure** -- a spider web is a network with no single
  center, built from connections rather than containers. This maps onto
  hypertext with remarkable fidelity: documents link to documents, and
  you traverse the structure by following strands (links). The metaphor
  encodes the key architectural insight of the World Wide Web -- that
  information is organized by connection, not by hierarchy. This was
  radical in 1989, when most information systems were hierarchical
  (directories, card catalogs, Gopher menus).
- **Universality of access** -- a web connects everything it touches.
  Any strand leads to any other strand given enough traversals. This
  maps onto the web's design principle: any document can link to any
  other document, regardless of who created it or where it is hosted.
  The metaphor captures the promise of universal interconnection in a
  single image.
- **Fragility and resilience** -- spider webs are both fragile (a
  single broken strand weakens the whole) and resilient (the structure
  can be rebuilt, and damage to one section does not destroy the whole).
  This maps onto hypertext: broken links are ubiquitous (link rot), but
  the web as a whole persists and regenerates. The metaphor encodes both
  properties without contradiction.

## Where It Breaks

- **Webs are traps** -- a spider builds a web to catch prey. The
  strands are sticky. Creatures that enter do not leave. This predatory
  function is entirely absent from the metaphor as Berners-Lee intended
  it, but it has become accidentally relevant. Attention-economy
  platforms design their products to be sticky, to trap users in
  engagement loops, to make leaving difficult. The web-as-trap metaphor
  that Berners-Lee did not intend may be the most accurate description
  of what the web became.
- **Webs have a single architect** -- a spider builds its web alone,
  according to its own design. The World Wide Web is built by millions
  of independent actors with no coordinating architect. The metaphor
  implies a coherent design where there is only emergent structure.
  This is why conspiracy theories about who "controls the web" are so
  persistent: the metaphor implies a builder, and people look for one.
- **The web is not the internet** -- the metaphor was so successful
  that it consumed the identity of its host. Most non-technical users
  use "the web" and "the internet" interchangeably, even though the
  web (HTTP/HTML) is one application layer running on the internet
  (TCP/IP). Email, FTP, DNS, and streaming protocols are not "the web,"
  but the metaphor's dominance has made them invisible. The web metaphor
  ate the internet.
- **Spiders are solitary** -- the social web (social media, wikis,
  collaborative platforms) maps poorly onto spider behavior. Spiders do
  not build webs together. The communal, collaborative nature of Web 2.0
  has no correspondent in the source domain. The metaphor was dead long
  before this mismatch became relevant, so nobody noticed.

## Expressions

- "Website" -- a location on the web, using spatial metaphor (site as
  place) layered on top of the web metaphor; nobody pictures a spider
  when they visit a website
- "Web page" -- a document on the web, importing the page metaphor from
  print publishing into the web metaphor; a page in a web is an odd
  image that nobody processes
- "Web browser" -- the tool for traversing the web, using a browsing
  metaphor (casually looking through a shop) layered onto the web
  metaphor; you browse a web as you might browse a store
- "Webmaster" -- the administrator of a website, a title that implies
  mastery of the web as a spider masters its web; largely obsolete,
  replaced by role-specific titles
- "Web crawler" -- a program that traverses the web by following links,
  the most arachnid term in active use; crawlers literally move along
  strands like spiders
- "The dark web" -- the portion of the web not indexed by search
  engines, using darkness to imply concealment; the metaphor suggests
  parts of the web that light (indexing) does not reach
- "Caught in the web" -- the trap meaning, resurfacing in critiques
  of internet addiction and attention capture

## Origin Story

Tim Berners-Lee, a physicist at CERN, wrote a proposal in March 1989
titled "Information Management: A Proposal." His supervisor famously
annotated it "Vague but exciting." Berners-Lee needed a name for his
system of hyperlinked documents accessible via the internet.

He considered several alternatives. "Information Mesh" was rejected as
too similar to "mess." "Mine of Information" was rejected because the
acronym (MOI) was too egocentric in French. "The Information Mine"
(TIM) was rejected for the same reason in English. "World Wide Web"
survived because the alliteration was memorable and the web metaphor
captured the interconnected, non-hierarchical topology he envisioned.

The first web server went live at CERN in December 1990. The first
website outside Europe launched in December 1991 at SLAC (Stanford
Linear Accelerator Center). By 1993, with the release of the Mosaic
browser, the web was growing exponentially. By 1995, the metaphor
was dead. "The web" was not a figure of speech; it was a proper noun.

The abbreviation "www" (World Wide Web) is one of the few abbreviations
that takes longer to say than the phrase it abbreviates -- nine
syllables versus three. This suggests the full metaphor was already
unnecessary; people just needed a short label, not an image.
