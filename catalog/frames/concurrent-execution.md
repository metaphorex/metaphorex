The simultaneous or interleaved execution of multiple computational tasks
within a system. Concurrent execution introduces problems that sequential
programming does not face: shared state must be coordinated, resources
must be allocated fairly, and tasks must be prevented from interfering
with each other. The domain borrows metaphors from textiles (threads,
weaving), social coordination (locks, semaphores, barriers), and
biological life cycles (spawning, sleeping, dying) to describe
fundamentally abstract scheduling and state-management problems.
